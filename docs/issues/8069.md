# #8069: different behavior when opt `fsqrt` with negative number in arm64 and x64

- **URL:** https://github.com/WebAssembly/binaryen/issues/8069
- **Author:** HerrCai0907
- **Created:** 2025-11-26
- **Updated:** 2025-11-26

## Description

`Literal` will use `std::sqrt` to calculate `f32.sqrt` and `f64.sqrt`. It cause output wasm different when we build in arm64 and x64.
